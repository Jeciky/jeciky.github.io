module.exports、exports、require和export、export default、import之前的区别与不同

发表于 2019-03-10 | 分类于 javascript

首先他们基于的规范就不同，module.exports、exports、require是基于CommonJS的模块规范，export、export default 、import是基于es6摸板规范的。

node中的module.exports、exports、require

Node应用由模块组成，采用CommonJS模块规范，在Node.js模块系统中，每个文件都视为独立的模块，拥有自己作用域，在一个文件里面定义的变量、函数、类相对于其他文件来说都是私有的，其他文件不可见。
对于上面的特性当我们就需要引用其他模块里类、函数、变量或者输出当前模块的类、函数、变量来让模块互相连接起来的时，就需要用到module.exports、exports、require。
require：用于引入模块、JSON、或本地文件。遵循以下规则：

以 ‘/‘ 为前缀的模板文件是绝对路径。
以 ‘./ ‘ 为前缀的模板是相对路径，相对于调用require的文件 。
当没有 ‘/‘ 、’./‘ 、’../‘ 开头来表示文件的时候，这个模块必须是核心模块或加载自node_modules目录
例如我们在开发中，用require(‘模块名’) 来加载某个模块时。这个时候会自动去node_modules目录中去找该模块的文件夹，那么他是怎么找到的，细心的同学可能会发觉，我们通过npm安装依赖的包都有个package.json文件，里面都有name和main属性，这个name：你要加载的文件名,main：这个包或者模块对应的主要文件，或者入口文件。当我们需要将一个目录作为模块时，可以在这个目录下新建package.json文件，并制定main模块。

module.exports与exports:用于指定一个模块导出的内容。module.export的赋值必须是立即完成的，不能在任何回调中完成。
module.exports与exports的不同：exports是module.exports的快捷方式，既exports=module.exports。

运用exports导出的方法会在模板执行之前赋值给module.exports，如果exports赋值呢新值，则他将不再绑定到module.exports,可以根据下面的这个例子进一步理解

ES6中的export、export.default、import

exoport、export.default、import是在ESMA6中提出的javascript模块体系，使得程序在编译时就能确认模块间的依赖关系，以及输入和输出变量。

export:用于规范模块的对外接口，一个模块就是一个独立的文件，外部无法获取，需要通过export输出变量或者方法

++注意：export命令规定的是对外的接口，必须与模板内部的变量建立一 一对应关系。++

与node的输出不同的是，export语句输出的接口，与其对应的值是动态绑定关系,既通过该接口，可以取到模板内部实时的值。

import:使用export命令定义的模板对外接口，通过import命令加载这个模块。
import的特性：

1：import是静态执行，所以不能使用表达式和变量，这些只能在运行时才能得到结果的语法结构。
2：import语句会自动提升到整个模块的头部，首先执行。
3：import命令输入的变量是只读的。
4：多次导入同一个模块只执行一次。

注：export default 在一个模块中只能出现一次。
